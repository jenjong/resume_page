Painting map by statistics

painting map

map('world', fill = TRUE, col = rainbow(30))
* `world` 데이터 베이스에는 국가별 경계에 대한 위-경도 정보가 있음
* `fill=TRUE` 는 국가별 경계의 안쪽을 색칠하는 옵션임
* `col = rainbow(30)` 은 색칠할 팔레트 정보임.
    + 무지개 색을 30개로 분할하여 사용함.
* 색깔이 칠해지는 순서는 `world` 데이터 베이스에 저장된 국가의 폴리곤 지표 순이며,
`rainbow(30)` 팔레트에서 색을 하나씩 가져와서 칠함.
    + 미국의 알래스카와 미국 색깔이 다름
* 30개 국가경계의 안쪽이가 칠해지고 난 후 31번째는 다시 첫번째 색을 가져와서 칠함

Unemployment rate map

* `unemp` 데이터는 county 식별번호(fips), 인구(pop), 실업률(enemp) 
데이터를 가지고 있다.

* `county.fips` 데이터는 county 식별번호(fips)를 가지고 있다. 
그리고 이 식별번호는 `county` 데이터베이스가 
가진 맵폴리곤 데이터의 그룹 인덱스(polyname)의 순서와 같다.


* `county` 는 지도위에 그림을 그릴 폴리곤 데이터 베이스다.

* `county`의 경계 안쪽을 칠하기 위해서 `county.fips` 식별번호 옆에 
필요한 통계량을 순서대로 정리하면 된다.

Unemployment rate map

unemp$colorBuckets <- as.numeric(cut(unemp$unemp, 
                                c(0, 2, 4, 6, 8, 10, 100)))
colorsmatched <- unemp$colorBuckets[match(county.fips$fips, unemp$fips)]
* `cut()` 함수를 이용해서 실업율(0,2,4,6,8,10,100)에 따라서 범주를 생성한다. 

* `colorsmatched`는 `fips`데이터가 몇 번째 범주에 속하는지  표시한다.
    + match 함수는 `county.fips$fips`에 들어 있는 값 하나하나가
unemp$fips에 몇 번째에 위치하는지 알려준다. 


    +  `unemp$colorBuckets[match(county.fips$fips, unemp$fips)]` 를 실행하면
    우리가 사용하게 될 `unemp$colorBuckets`의 값을 `county.fips$fips`의 순서에 
    맞춰 정렬하게 된다. 

    + unemp 데이터의 colorBuckets을 colorsmatched에 저장

Unemployment rate map

colors = c("#F1EEF6","#D4B9DA","#C994C7","#DF65B0","#DD1C77","#980043")
if(!require(mapproj)){install.packages("mapproj") ;library(mapproj)}
map("county", col = colors[colorsmatched], fill = TRUE,
    resolution = 0, lty = 0, projection = "polyconic")
* `col` 은 우리가 범주를 6개로 나누었으므로 6개로 잡는다.
* `map` 함수의 `col`과 `fill=TRUE`  옵션을 이용해서 county별 실업률을 도식한다.
* `resolution=0` 는 `county` DB 경계정보를 모두 사용한다.
* `lty=0` 을 사용해 경계를 표시하지 않는다.
* `projection` 은 구면좌표를 평면으로 정사형하는 방법을 결정한다. 
(See mapproject {mapproj})

Pie chart on map

  • 지도 위에 원하는 위치에 새로운 그래프를 도식할 수 있다.
  • 여기서는 특정 위치에 pie-chart를 그리는 시각화 방법을 소개한다.

Pie chart on map

if(!require(mapplots)){install.packages("mapplots") ;library(mapplots)}
if(!require(ggmap)){install.packages("ggmap") ;library(ggmap)}
if(!require(mapdata)){install.packages("mapdata") ;library(mapdata)}

map('worldHires', 'South Korea')
seoul_loc = geocode('seoul')
busan_loc = geocode('Busan')
add.pie(z=1:2,labels = c('a','b'), 
        x = seoul_loc$lon, y = seoul_loc$lat, radius = 0.5)
add.pie(z=4:3,labels = c('a','b'),
        x = busan_loc$lon, y = busan_loc$lat, radius = 0.5)
    * `ggmap` 패키지의 `geocode` 함수를 이용하여 서울과 부산의 위경도 좌표를 받음
    * 저장된 위경도 좌표는 데이터프레임으로 위도(lat), 경도(log) 열을 가지고 있다
    * `mapplots` 패키지의 `add.pie` 함수를 이용해 지도위에 직접 pie chart를 생성
    * `radius` 옵션은 pie chart의 크기를 정해준다.
